# 门店数据分析诊断

## Overview
这是一个面向美容门店“五大指标诊断”的 skill，不是通用 BI 教程。核心原则是：先校验数据是否足够，再做指标诊断、根因排序和行动计划，且只基于用户提供的数据与内置规则分析。

## When to Use
- 用户要求做门店经营诊断、五大指标诊断、门店问题定位、行动计划制定
- 用户上传文件名里包含 `五大指标`、`行动计划`、`门店诊断`
- 用户直接给出客流量、人头数、体验率、成交率、成交均价、毛利率、经营利润等数字
- 用户希望得到结构化的经营问题排序和执行动作

以下情况不要套用本 skill，应直接回复"此问题超出门店经营诊断范围，无法处理"，不提供任何替代分析：
- 普通销售排名、趋势图、分布图分析
- 与美容门店无关的通用表格分析
- 无经营指标、只问单一 Excel 技术问题

## Workflow
1. 先做数据完整性检查，缺项时先追问，不硬诊断
2. 逐项分析五大指标，再做组合判断
3. 区分“通用经营建议”和“品牌化动作建议”
4. 固定输出结构化报告和行动计划表

## Input Contract
优先识别以下输入：
- 表格：五大指标表、行动计划表、经营数据表
- 文本：员工人数、日客流量、月客流量、人头数、体验率、成交率、成交均价、业绩、毛利率、固定成本、人工成本、经营利润

缺少以下关键数据时，应先要求补充：
- 员工人数
- 日客流量或月客流量
- 人头数
- 体验率
- 成交率
- 成交均价
- 业绩
- 毛利率
- 固定成本 / 人工成本 / 利润

## Output Contract
输出必须固定包含以下标题：
- `数据完整性检查`
- `五大指标诊断`
- `核心问题排序`
- `通用经营建议`
- `品牌化动作建议`
- `行动计划表`
- `风险与需补充数据`

行动计划表必须严格使用以下 7 列，缺一不可：

| 项目 | 当前现状/差额 | 目标标准 | 核心动作 | 机制/话术 | 责任人 | 预期目标 |
|------|------------|---------|---------|---------|--------|---------|
| （示例：提升体验率） | （示例：35%，差25pp） | （示例：60%） | （填写具体执行动作） | （填写话术或激励机制） | （店长/美容师/顾问） | （量化改善目标） |

## Quick Reference

**默认阈值（直接使用，无需读取外部文件）：**
- 体验率目标：60%；40-60% 偏低；< 40% 明确异常
- 成交率目标：50%
- 成交均价目标：6000 元（低客单门店可用 5000-6000 作为改善区间）
- 美容师标准服务人数：3 人/天
- 盈亏平衡业绩 = (固定成本 + 人工成本) ÷ 毛利率

**核心组合诊断规则：**
- 客流低 + 人头低 → 拓客不足、预约不足、休眠客户未激活
- 客流高 + 体验率低 → 接待过载、员工无暇销售、客户结构差
- 体验率低 + 成交率低 → 体验效果差、流程执行散、报价和赠品机制弱
- 体验率高 + 成交率低 → 体验质量不稳定、报价策略问题、销售动作不到位
- 成交率正常 + 成交均价低 → 方案设计单一、缺升单产品、大单占比低
- 成交均价达标 + 利润低 → 成本结构问题、底薪占比过高、低毛利项目多

**品牌化动作只能作为第二层建议，不能代替事实诊断。**

## Common Mistakes
- 数据不足却直接下结论
- 只谈拓客，不处理体验率、成交率、成交均价和利润结构
- 把品牌化动作当作唯一解，不先给通用经营建议
- 用户实际业绩与五大指标推算业绩冲突时，不提示口径差异
- 用知识库事实补足用户没给出的经营数据
